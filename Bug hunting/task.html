Description

It's possible that the author made some errors in typing and coding logic while developing the project, which could have impacted the program's functionality in certain instances.

Your task for this stage is to find and fix all the bugs related to the logic of processing user requests. You might want to test the program manually or write unit tests. You will likely find several cases of incorrect results.

As an example, a consequence of one of the bugs in the program is the result of the /and command execution for booleans data type:

Welcome to Data Memory!
Possible actions:
1. Memorize booleans
2. Memorize numbers
3. Memorize words
0. Quit
> 1
Perform action:
> /add true
Element false added
Perform action:
> /add false
Element true added
Perform action:
> /and 0 1

...

Hint #2

The extended table shows executable commands, data types, and the bug description of unexpected results:

Data type Command Description
Booleans, Numbers, Words /getRandom Always prints the first element in a list
Booleans, Numbers, Words /readFile Prints a new list size, not the number of imported elements
Booleans, Numbers, Words /mirror Doesn't modify the list, the result is ignored
Booleans, Numbers, Words /sort For the descending argument, execution is performed as ascending
Booleans /and Always performed as (list[i] && list[i])
Booleans /or Always performed as (list[j] || list[j])
Booleans /flip Always performed as !list[0]
Booleans /logShift Instead of the length of the list uses Byte.SIZE
Numbers /factorial Result of the calculation \(n!\) for n==0 and n==1 is incorrect
Numbers /average The average should be calculated as \({a_1+a_2+...+a_n \over n}\), not as \({a_1+a_2+...+a_n \over 2}\)
Numbers /sum, /subtract, /multiply, /sumAll, /pow, /average

Ignores possible integer overflow or underflow. Tests are performed under conditions in which the result is in the \((-2^{63},2^{63})\) range
Numbers /divide, /pow, /average Ignores the possibility of fractional numbers. In this case, the result can have up to six digits after the decimal point.
Words /join Always performed with the _ delimeter
Words /swapCase Equal to the /upperCase command
Objectives

Fixing all the bugs is the main goal for this stage. You might want to rewrite the initial code without modifying its use cases and feedback if you'd like to.
