Description

It's full of bugs and illegible, but we'll work to fix it. Based on the class descriptions, this program seems to be intended to store and analyze user input data in three categories: numbers, booleans, and words.

In this stage, the main task is to format the code according to coding style conventions and familiarize yourself with the initial code and its structure. We won't test these actions, but it is necessary to do them. In this stage, organize the code so you will feel comfortable working with it in the following stages.

The critical point is that we want to improve the program without changing it. Therefore, when working on stages, we can add innovations and change the structure of the code but do not modify its use cases and feedback.

You might want to read some hints as you go through the project. You can try to find problems in the code by yourself or get a clue. As the number of hints increases, so does the information needed to fix the problem.

Hint #2
